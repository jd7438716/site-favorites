# Site Favorites Sidebar

按域名分组的网页收藏夹：全站注入 content script，在页面右侧提供悬浮按钮与页面内侧边栏，快速收藏/搜索/编辑当前站点的链接。

核心定位是「域名级收藏夹」：解决两类场景——

- 网站本身没有收藏夹功能
- 不想/不能登录账号，但仍希望临时或长期保存该站点的常用入口

## 功能

- 右侧悬浮按钮（可上下拖动改变位置）
- 打开/关闭侧边栏
- 一键收藏/取消收藏当前页面
- 站点内收藏列表：按标题/URL 搜索、编辑、删除
- 支持将链接拖到列表区域快速添加
- 数据存储在 `chrome.storage.local`，按域名分组

## 安装（开发者模式）

1. 打开 Chrome：`chrome://extensions/`
2. 右上角开启「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择本项目目录 `chrome_extension1`
5. 打开任意网页，右侧会出现悬浮按钮

## 使用说明

- **打开侧边栏**：点击悬浮按钮上方的图标
- **收藏/取消收藏当前页**：点击悬浮按钮下方的书签图标
- **搜索**：在侧边栏顶部输入框搜索标题或 URL
- **编辑/删除**：在列表每条记录右侧操作按钮中进行
- **拖拽添加**：把浏览器地址栏链接或页面中的链接拖到列表区域即可添加
- **关闭侧边栏**：再次点击开关按钮，或按 `Esc`

## 权限说明

本扩展当前仅包含 content script，因此权限配置较少：

- `storage`：用于保存收藏数据与悬浮按钮位置
- `host_permissions: <all_urls>`：用于在所有网页注入侧边栏与按钮

## 数据结构

数据保存在 `chrome.storage.local`：

- `sf_bookmarks_by_domain_v1`：对象结构 `{ [domain: string]: Array<{ id, url, title, createdAt }> }`
- `sf_fab_pos_v1`：对象结构 `{ top: number }`，记录悬浮按钮距顶部位置

## 开发

当前为纯原生实现（无构建步骤）：

- 入口：`manifest.json`
- 注入脚本：`contentScript.js`

修改代码后，在 `chrome://extensions/` 中对该扩展点击「刷新」，再刷新目标网页即可生效。

## 已知限制

- 仅在页面内工作（没有独立的扩展页面/弹窗）
- 数据按 `location.hostname` 分组；同一站点不同协议/路径归为同组
